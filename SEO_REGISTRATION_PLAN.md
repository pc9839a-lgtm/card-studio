# SEO 등록 및 최적화 기획안

## 1. 목적

- 이 프로젝트를 검색엔진에 정상 등록한다.
- `온라인 명함 만들기`, `명함 템플릿`, `디지털 명함 제작`, `QR 명함 만들기` 같은 검색 의도에 맞는 노출 기반을 만든다.
- 현재의 웹앱 UI를 크게 해치지 않으면서도 검색엔진이 읽을 수 있는 정적 콘텐츠와 메타 구조를 추가한다.

## 2. 현재 상태 진단

### 현재 확보된 것

- `robots.txt` 존재
- `sitemap.xml` 존재
- 단일 페이지 기준 정적 HTML 문서 존재
- `h1` 존재
- footer와 기본 브랜드 정보 존재

### 현재 부족한 것

- `<title>`이 한국어 검색 의도를 거의 반영하지 못함
- `meta description` 없음
- `canonical` 없음
- Open Graph / Twitter 메타 없음
- JSON-LD 구조화 데이터 없음
- 검색용 정적 소개 본문이 부족함
- manifest 텍스트 일부 깨짐
- 실서비스 도메인 확정 전 기준 URL이 `pages.dev`에 묶여 있음

## 3. SEO 목표 구조

이 프로젝트는 단순 앱 화면만 두는 방식보다 아래 구조가 맞다.

1. 상단: 현재 명함 제작 앱
2. 하단: 검색용 랜딩 콘텐츠
3. head: 검색 노출용 메타 태그 / canonical / structured data
4. 루트 파일: robots / sitemap / manifest / favicon 정리

즉, `앱 + 랜딩페이지` 구조로 가는 것이 핵심이다.

## 4. 핵심 키워드 방향

### 1차 핵심 키워드

- 온라인 명함 만들기
- 명함 템플릿
- 디지털 명함 제작
- QR 명함 만들기
- 명함 PNG 저장

### 2차 보조 키워드

- 모바일 명함 제작
- 무료 명함 디자인
- 실시간 명함 미리보기
- 양면 명함 제작
- 명함 PDF 저장

### 카피 방향

- 브랜드명만 전면에 두지 않고 기능 키워드를 같이 노출한다.
- 영어 브랜드명은 유지하되, 제목과 설명은 한국어 중심으로 설계한다.
- 과장 표현보다 실제 기능 중심으로 작성한다.

## 5. 수정 범위

### 파일 단위

- `index.html`
- `style.css`
- `site.webmanifest`
- `robots.txt`
- `sitemap.xml`
- 필요 시 썸네일용 이미지 자산 추가

### 수정 원칙

- 현재 UI 흐름은 유지
- 검색용 콘텐츠는 앱 하단에 별도 섹션으로 추가
- 검색엔진이 JS 실행 전에도 읽을 수 있는 텍스트를 HTML에 직접 포함
- 모바일/PC 레이아웃을 깨지 않게 최소 침범 방식으로 추가

## 6. 패치 단계

## Stage 1. 등록 최소 조건 패치

목표:
- 검색엔진 등록 가능한 최소 기반 확보

작업:
- `<title>` 재작성
- `meta description` 추가
- `meta robots` 추가
- `canonical` 추가
- Open Graph 추가
- Twitter 카드 추가
- favicon 링크 명시
- `theme-color` 명시
- manifest 깨진 텍스트 수정

완료 기준:
- head 메타 구조가 기본 SEO 형식을 만족
- 공유 링크 미리보기 기본 대응 가능

## Stage 2. 검색용 정적 콘텐츠 패치

목표:
- 검색엔진이 읽을 수 있는 실제 본문 확보

작업:
- 앱 하단에 소개 섹션 추가
- 주요 기능 섹션 추가
- 사용 대상 섹션 추가
- 저장/출력 특징 섹션 추가
- FAQ 섹션 추가

완료 기준:
- 초기 HTML 기준으로 의미 있는 설명 텍스트가 충분히 존재
- 검색 키워드와 실제 기능 설명이 연결됨

## Stage 3. 구조화 데이터 패치

목표:
- 검색엔진이 페이지 성격을 더 정확히 이해하도록 지원

작업:
- `SoftwareApplication` JSON-LD 추가
- `Organization` JSON-LD 추가
- `WebSite` JSON-LD 추가
- FAQ를 실제로 넣을 경우 `FAQPage` JSON-LD 검토

완료 기준:
- Rich Results Test 기준 기본 구조 검증 가능 상태

## Stage 4. 색인 등록 준비 패치

목표:
- 실제 도메인 기준으로 등록 가능한 상태 정리

작업:
- 실서비스 도메인 확정
- `canonical` 도메인 반영
- `robots.txt`의 sitemap 경로 반영
- `sitemap.xml`의 `<loc>` 도메인 반영
- favicon / manifest / og 이미지 주소 반영

완료 기준:
- 모든 URL 기준이 한 도메인으로 통일됨

## Stage 5. 검색엔진 등록

목표:
- 실제 색인 수집 시작

작업:
- Google Search Console 등록
- Google sitemap 제출
- Naver Search Advisor 등록
- Naver sitemap / robots 확인

완료 기준:
- 주요 검색엔진에서 사이트 소유 확인과 sitemap 제출 완료

## 7. 권장 메타 구조

### title 초안

- 온라인 명함 만들기 | Business Card Studio
- 명함 템플릿 제작 및 PNG 저장 | Business Card Studio

### description 초안

- 템플릿 선택부터 회사명, 로고, 이미지, QR 설정까지 한 번에 가능한 온라인 명함 제작 도구. 모바일과 PC에서 실시간 미리보기 후 PNG와 PDF로 저장할 수 있습니다.

### canonical

- 실서비스 도메인 루트 URL 1개만 사용

## 8. 랜딩 콘텐츠 구성안

### 섹션 A. 서비스 소개

- 온라인에서 바로 명함을 만들 수 있는 도구라는 점
- 설치 없이 사용 가능하다는 점

### 섹션 B. 핵심 기능

- 템플릿 추천
- 실시간 미리보기
- 앞면/뒷면 개별 편집
- 로고/이미지/QR 설정
- PNG / PDF 저장

### 섹션 C. 사용 대상

- 프리랜서
- 소상공인
- 1인 브랜드
- 디자이너
- 영업/상담 인력

### 섹션 D. FAQ

- 무료 사용 여부
- 저장 포맷
- 모바일 사용 가능 여부
- QR 생성 가능 여부
- 양면 명함 지원 여부

## 9. 비기능 체크리스트

- 주요 텍스트가 JS 없이도 HTML에 존재하는가
- 모바일에서 추가된 SEO 섹션이 지나치게 길거나 UI를 깨지 않는가
- favicon / manifest / 공유 이미지 링크가 깨지지 않는가
- robots가 잘못 막고 있지 않은가
- sitemap URL이 실제 배포 주소와 일치하는가

## 10. 리스크

### 리스크 1. 앱 중심 구조

- 지금은 도구 UI 비중이 크고 랜딩 본문이 약해서 색인만 되고 노출은 약할 수 있음

### 리스크 2. 도메인 변경

- `pages.dev`로 먼저 등록 후 커스텀 도메인으로 바꾸면 canonical / sitemap 재정비가 필요함

### 리스크 3. 자바스크립트 의존

- 검색엔진이 JS 렌더링 없이도 읽을 수 있는 정적 설명을 반드시 확보해야 함

## 11. 바로 진행할 1차 범위

1. `index.html` head SEO 메타 정리
2. `site.webmanifest` 깨진 텍스트 수정
3. 앱 하단 소개 섹션 추가
4. FAQ 섹션 추가
5. JSON-LD 추가

이 1차까지 끝난 뒤 검색 등록을 진행하는 순서가 맞다.
