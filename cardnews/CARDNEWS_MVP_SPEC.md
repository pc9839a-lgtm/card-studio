# CardStudio 카드뉴스 제작기 MVP 명세서

## 1. 목적

- 기존 명함 제작기와 별개로 카드뉴스 제작 기능을 추가한다.
- 페이지 분리는 나중에 하고, 우선 같은 프로젝트 안에서 카드뉴스 편집 기능을 먼저 구현한다.
- 1차 목표는 `간단하지만 실제로 쓸 수 있는 카드뉴스 제작기`를 만드는 것이다.

## 2. 원칙

- 기존 명함 기능은 최대한 건드리지 않는다.
- 카드뉴스 기능은 새 상태 구조와 새 UI로 추가한다.
- 복잡한 디자인 툴이 아니라 `빠르게 완성 가능한 제작기`를 목표로 한다.
- 1차는 `다중 슬라이드 전체 편집기`가 아니라 `단일 장 카드뉴스 제작기` 중심으로 간다.
- 추후 여러 장 카드뉴스 확장은 2차에서 진행한다.

## 3. 1차 MVP 범위

### 반드시 포함

- 캔버스 비율 선택
  - 정사각형 `1080 x 1080`
  - 세로형 `1080 x 1350`
- 기본 템플릿 선택
- 타이틀 입력
- 서브텍스트 입력
- 타이틀 위치 조정
- 타이틀 크기 조정
- 타이틀 색상 변경
- 배경색 변경
- 배경 이미지 업로드
- 일반 이미지 업로드
- 오버레이 색상/투명도
- 간단 도형 추가
  - 사각형
  - 원
  - 선
- 레이어 순서 변경
- PNG 저장

### 1차에서 제외

- 여러 장 슬라이드 관리
- 팀 협업
- 애니메이션
- 텍스트 효과 고도화
- SVG 자유 편집
- 자동 정렬 엔진
- ZIP 일괄 저장
- AI 자동 생성

6. 레이어 순서 조정

## 6. 캔버스 포맷

### 정사각형

- 작업 기준: `1080 x 1080`
- 인스타 게시물 / 카드형 홍보물 기준

### 세로형

- 작업 기준: `1080 x 1350`
- 인스타 세로형 카드뉴스 기준

## 7. 템플릿 방향

템플릿은 완성 디자인보다 `배치 프리셋` 중심으로 설계한다.

### 템플릿 1. 텍스트 중심형

- 상단 타이틀
- 하단 서브텍스트
- 배경색 중심

### 템플릿 2. 이미지 중심형

- 상단 텍스트
- 하단 큰 이미지

### 템플릿 3. 커버형

- 중앙 타이틀
- 전체 배경 이미지
- 강한 오버레이

### 템플릿 4. 리스트형

- 제목
- 포인트 문구 3개
- 구획 도형 포함

### 템플릿 5. 강조 문구형

- 짧은 헤드라인
- 큰 숫자 또는 키워드
- 강조색 중심

## 9. 편집 우선순위

### 텍스트

가장 중요하다.

- 제목
- 본문
- 색상
- 크기
- 위치

### 이미지

두 번째로 중요하다.

- 배경 이미지
- 메인 이미지

### 도형 / 오버레이

디자인 완성도를 올리지만 1차에서는 최소 기능으로 충분하다.

## 10. 저장/내보내기

### 1차 MVP

- 현재 장 PNG 저장

### 2차

- 프로젝트 저장
- 장별 복제
- 양식 저장
- 다중 장 ZIP 저장

## 11. 기술 구현 방향

### 권장 파일

- `card-news.html` 또는 임시 마운트용 섹션
- `card-news.js`
- 필요 시 `card-news.css`

### 상태 처리

- 명함 상태와 분리
- 새 전역 상태 객체 사용
- 이미지와 도형은 레이어 배열로 관리

### 렌더링

- DOM 기반 미리보기
- 기존 export 방식을 참고하되 카드뉴스용 캔버스 기준으로 재구성

## 12. 레이어 모델

레이어는 아래 순서를 기본값으로 가진다.

1. 배경색
2. 배경 이미지
3. 도형
4. 일반 이미지
5. 오버레이
6. 텍스트

사용자는 특정 레이어를 선택하고 순서를 바꿀 수 있어야 한다.

## 13. 모바일 편집 흐름

모바일은 명함 제작기처럼 순차형 편집이 유리하다.

### Step 1

- 비율 선택

### Step 2

- 템플릿 선택

### Step 3

- 제목/본문 입력

### Step 4

- 배경/이미지 설정

### Step 5

- 도형/오버레이 설정

### Step 6

- 저장

## 14. 데스크톱 편집 흐름

데스크톱은 한 화면 고정 편집형이 좋다.

- 좌측 패널에서 섹션 열기
- 우측 캔버스 실시간 반영
- 하단 저장 버튼

## 15. 리스크

### 리스크 1. 레이어 순서 구현

- 단순 위치 조정만 있는 명함보다 상태가 복잡해짐

### 리스크 2. 이미지/배경 이미지 충돌

- 동일한 업로드 로직이지만 역할이 다르므로 분리 필요

### 리스크 3. 템플릿 설계 과다

- 템플릿을 너무 많이 만들면 구현이 지연됨

### 해결

- 1차는 4~5개 템플릿만
- 레이어는 최소 타입만
- 슬라이드는 단일 장만

## 16. 1차 개발 범위 확정

다음 기능까지만 구현하면 MVP로 인정한다.

1. 정사각형 / 세로형 선택
2. 템플릿 5종
3. 타이틀 / 본문 입력
4. 타이틀 색상 / 크기 / 위치 조정
5. 배경색 / 배경 이미지
6. 메인 이미지 업로드
7. 간단 도형 3종
8. 오버레이
9. 레이어 순서
10. PNG 저장
